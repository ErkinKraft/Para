## Клавиатура по сети - быстрый шэринг нажатий клавиш

Современное настольное приложение на PySide6 (Qt), которое позволяет передавать нажатия клавиш с одного ПК на другой по сети в реальном времени. Подходит для совместной работы, удалённых демо, помощи друзьям и быстрой эмуляции клавиатуры без долгой настройки удалённого доступа.

### Возможности
- **Два режима**: Отправлять (capture) и Принимать (inject)
- **Мгновенная передача**: низкая задержка за счёт TCP_NODELAY и легкого JSONL-протокола
- **Простой старт**: минимальный интерфейс, автоматический выбор порта у принимающей стороны
- **Совместимость с Windows** (разработано и протестировано под Windows 10/11)

### Как это работает
- Отправитель перехватывает нажатия с клавиатуры у себя локально и кодирует их в компактные JSON-сообщения.
- Получатель принимает эти события по TCP и эмулирует те же нажатия у себя.
- Для минимальной задержки используется выключение Nagle (TCP_NODELAY) и строчный фрейминг (по переводу строки).

Протокол событий (пример):
```
{"t":1,"k":"char","v":"a"}     # нажата клавиша 'a'
{"t":0,"k":"char","v":"a"}     # отпущена клавиша 'a'
{"t":1,"k":"key","v":"enter"}  # нажат Enter
{"t":0,"k":"key","v":"enter"}  # отпущен Enter
```

### Требования
- Windows 10/11
- Python 3.10+

### Установка
Запустите **start.bat**:

```powershell
       [1] Запустить
       [2] Установить зависимости
       [3] Выход

Выберите действие (1-3): 2
```

Выбираем пункт 2 и жмём enter и ждём пока не вылетит

### Запуск
1) Запустите start.bat:

```powershell
       [1] Запустить
       [2] Установить зависимости
       [3] Выход

Выберите действие (1-3): 1
```

2) Выберите режим:
- Принимать: приложение покажет случайный порт и автоматически начнет ожидание подключения.
- Отправлять: введите IP и порт принимающей стороны, нажмите «Подключиться».

3) После успешного подключения:
- В окне появится зелёный индикатор с галочкой.
- Через ~1 секунду окно свернётся, а передача/приём клавиш продолжится в фоне.

### Сетевые заметки
- Приложение ориентировано на локальную сеть (LAN) для минимальной задержки.
- Убедитесь, что брандмауэр Windows разрешает входящие/исходящие соединения для Python.
- Принимающая машина должна сообщить отправителю свой IP и порт, показанный в приложении.

### Безопасность
- Соединение без шифрования и аутентификации, предназначено для доверенных сетей.
- Не используйте в общедоступных/недоверенных сетях. Для интернета добавьте VPN или туннель с шифрованием.

### Частые проблемы
- Не устанавливаются зависимости: обновите pip (`python -m pip install --upgrade pip`) и повторите установку.
- Нет эмуляции на принимающей стороне: запустите PowerShell/CMD «От имени администратора».
- Не подключается: проверьте IP/порт и правила брандмауэра, а также что принимающее приложение запущено и слушает порт.

### Структура проекта
- `main.py` - Qt GUI, выбор режима, подключение, индикация статуса
- `sender.py` - захват клавиш (`pynput`) и отправка по TCP
- `receiver.py` - приём событий и эмуляция клавиш на целевой машине
- `common.py` - утилиты: низколатентные сокеты, JSONL-протокол
